---

## 1. Context

MAP-6P established that the Build 42 candidate mod loads without a
candidate-specific crash, but the candidate spawn region was not visible in the
spawn selection screen. This protocol defines the human inspection steps to
diagnose the activation gap.

The likely causes in priority order:
1. Server preset `Map=` does not include the candidate map.
2. `spawnregions.lua` is missing from the mod or has the wrong format.
3. Server `_spawnregions.lua` does not reference the candidate.
4. `WorkshopItems=` mismatch or `Mods=` line incomplete.

---

## 2. Verify installed mod folder layout (human inspection)

Open PowerShell and run each check manually. These are read-only operations.

```powershell
# Check that the mod folder exists with the correct 42/ layout
$mod = 'C:\Users\Palmacede\Zomboid\mods\pzmapforge_build42_candidate_001_test_clean'
Test-Path "$mod\42\mod.info"
Get-Content "$mod\42\mod.info" | Select-String 'id='
```

Expected: `id=pzmapforge_build42_candidate_001`

```powershell
# Check map files inside the 42/ layout
$mapDir = "$mod\42\media\maps\pzmapforge_build42_candidate_001"
Test-Path "$mapDir\map.info"
Test-Path "$mapDir\spawnpoints.lua"
Test-Path "$mapDir\spawnregions.lua"
Test-Path "$mapDir\objects.lua"
Test-Path "$mapDir\0_0.lotheader"
Test-Path "$mapDir\world_0_0.lotpack"
Test-Path "$mapDir\chunkdata_0_0.bin"
```

Record which files return True and which return False. The presence of
`spawnregions.lua` is critical for spawn activation.

---

## 3. Verify spawnregions.lua content (human inspection)

If `spawnregions.lua` exists, inspect it:

```powershell
Get-Content "$mapDir\spawnregions.lua"
```

Expected content (from MAP-6M packet template):

```lua
function SpawnRegions()
    return {
        {name="PZMapForge Candidate Cell", file="media/maps/pzmapforge_build42_candidate_001/spawnpoints.lua"},
    }
end
```

If the file is absent or contains a different format, the spawn region will
not appear. The MAP-6M packet provides `pzmapforge_candidate_spawnregions.lua`
which should be copied to `spawnregions.lua` at the correct path.

---

## 4. Verify server preset files (human inspection)

The server ini controls which maps are active. Inspect the server config files:

```powershell
$serverName = 'PZMF_B42_CANDIDATE_CLEAN_001'
$serverDir  = 'C:\Users\Palmacede\Zomboid\Server'
Test-Path "$serverDir\$serverName.ini"
Test-Path "$serverDir\${serverName}_spawnregions.lua"
```

If the `.ini` file does not exist, the server preset was not created correctly.
Create it from the Host menu before the test, or create it manually.

---

## 5. Inspect server ini Mods, Map, WorkshopItems lines (human inspection)

```powershell
$serverName = 'PZMF_B42_CANDIDATE_CLEAN_001'
$iniPath    = "C:\Users\Palmacede\Zomboid\Server\$serverName.ini"
Get-Content $iniPath | Select-String -Pattern '^Mods=|^Map=|^WorkshopItems='
```

Record the exact values of each matching line.

Required for candidate activation:
- `Mods=` must include `pzmapforge_build42_candidate_001`
  (or the mod folder name used during install)
- `Map=` must include `pzmapforge_build42_candidate_001`

Example of correct Map= line:
```text
Map=pzmapforge_build42_candidate_001;Muldraugh, KY
```

Note: `Muldraugh, KY` is the default vanilla map entry. The candidate map must
be added alongside it (separated by `;`), not replacing it.

If `Map=` does not include the candidate map ID, the spawn region will not
appear even if all binary files are present.

---

## 6. Inspect server spawnregions file (human inspection)

```powershell
$serverName = 'PZMF_B42_CANDIDATE_CLEAN_001'
$srPath = "C:\Users\Palmacede\Zomboid\Server\${serverName}_spawnregions.lua"
if (Test-Path $srPath) { Get-Content $srPath }
```

The server spawnregions file should reference:
```text
media/maps/pzmapforge_build42_candidate_001/spawnpoints.lua
```

If this file is absent or does not reference the candidate path, the spawn
region will not appear on the server's spawn selection screen.

## 7. Fill the diagnostic record

Use the record template generated by
`scripts/prepare-map6p-spawn-activation-diagnostic.ps1` to record the findings:

| Field | Value to record |
|---|---|
| mod_folder_exists | yes / no |
| mod_info_id_matches | yes / no |
| map_info_exists | yes / no |
| spawnpoints_exists | yes / no |
| spawnregions_in_mod_exists | yes / no |
| server_ini_exists | yes / no |
| mods_line_contains_candidate | yes / no |
| map_line_contains_candidate | yes / no |
| server_spawnregions_exists | yes / no |
| server_spawnregions_references_candidate | yes / no |
| candidate_spawn_region_visible | no (MAP-6P observation) |

After identifying the gap, fix the specific issue and re-run the MAP-6O clean
retest protocol. Record the result in a new session (MAP-6Q or equivalent).
